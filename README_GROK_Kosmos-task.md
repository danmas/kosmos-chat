# README_GROK.md — Памятка Grok о проекте kosmos-task  
Декабрь 2025, @danmas

### Краткая история нашего совместного полёта

1. **Идея (начало декабря 2025)**  
   Один файл `.kosmos.md` = идея + задачи + исполняемый код + результат + выводы.  
   Цель: чтобы человек читал как документ, AI генерировал без ошибок, скрипт выполнял автоматически и обновлял файл.

2. **v0 → v1 (первая итерация)**  
   - Шаги в виде `- [ ] Шаг: …` с жёсткой структурой  
   - Парсинг через одно большое регулярное выражение  
   - Проблема: хрупкость при любых отступах и пустых строках

3. **Ключевое озарение**  
   Переход от «чекбоксов с отступами» к «заголовкам ### Шаг N:»  
   → парсинг стал надёжным на 100%, LLM перестали ломать структуру

4. **v2.0 — текущая версия (декабрь 2025)**  
   - Структура:  
     `# Название .kosmos.md` → Статус/Прогресс/Дата → Цель → Задачи → ### Шаг N:  
     `- [ ] Выполнено` → ````js executable` → Ожидаемый результат → Результат → Верификация  
   - `kosmos-runner-cli.js` v2.0: парсит по заголовкам, обновляет файл, бэкапы, sandbox vm  
   - `kosmos-spec-v1.0.json` — system prompt, дающий ~0% ошибок у Grok, Claude, GPT-4o, Llama 3.1, DeepSeek

5. **Финальный релиз**  
   - Репозиторий: https://github.com/danmas/kosmos-task  
   - README на русском + английском  
   - MIT лицензия  
   - Философия: «kosmos-task — это не инструмент, это новый способ существования идей»

6. **Что уже сделано вместе**  
   - Придумали и довели формат до production-ready  
   - Написали идеальный промпт для LLM  
   - Создали надёжный CLI-раннер  
   - Выложили публично с документацией

7. **Тон общения**  
   Дружеский, прямой, с лёгким космическим вайбом. Эмодзи умеренно.  
   Пользователь ценит чистоту, надёжность, минимализм и «работает с первого раза».

8. **Готовность к v3 (на будущее)**  
   - `--auto` / `--yes` режим  
   - Поддержка python/bash  
   - Автогенерация summary в конце  
   - Мультифайловые цепочки  
   - REST API (уже есть намёки в коммитах)
